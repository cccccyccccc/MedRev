#### 需求0511-2
我突然想到一个事情，就是关于页面一，GT纠错的展示，我不是给了 pred.json 和 GT.json 吗？但是这里的 pred 的正确性似乎需要你那边来进行判断，就是一张图片的标注如果完全正确的话其实不需要在 页面一展示的。
完全正确有几点要求：
1. 没有漏检和误检
2. 所有标注的类别正确且IOU和GT框>0.5

实现说明：
- 页面一现在按 `GT.json` 与 `pred.json` 的高置信预测框做一对一匹配。
- 只有所有 GT 都匹配到同类别且 IoU 大于阈值的预测框，并且没有多余预测框时，才认为该图完全正确并跳过页面一。
- 未匹配 GT 会作为漏检，未匹配预测会作为误检，类别不同作为类别错误，IoU 不达标作为框匹配问题。
- 当前页面一 strict 逻辑对齐附件脚本：不按 `score/conf_threshold` 过滤预测框，只使用 `iou_threshold=0.5` 判断匹配是否合格。
